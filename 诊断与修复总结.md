### 4. 发现的问题 🔴

**核心问题**: `send_command` 方法会停止设备输出

```python
# 问题代码 (serial_client.py 第232-274行)
def send_command(self, command: str, wait_response: bool = True):
    # ...
    # 先停止输出，避免数据干扰
    self._send_stop()  # ❌ 这会停止设备的连续输出!
    time.sleep(0.15)
    # ...
```

**影响**: 
- 连接过程中调用任何命令都会停止设备输出
- 可能导致设备无法进入连续输出模式

**次要问题**:
- 连接流程等待时间不足
- 缺少原始数据日志,难以诊断

2. **运行主程序**:
   ```bash
   python main.py
   ```
   观察日志输出,确认数据流

3. **观察关键日志**:
   ```
   [DMT143] <<< 原始数据(...)
   [DMT143] <<< 数据行: ...
   [DMT143] >>> 解析成功: {...}
   ```
   如果看到这些日志,说明数据接收和解析正常

### 如果仍不显示数据
1. **检查硬件**:
   - 设备是否通电
   - 串口线是否连接正确
   - 设备是否被其他程序占用

2. **查看日志**:
   - 是否看到 "原始数据" 日志 → 判断是否收到数据
   - 是否看到 "解析成功" 日志 → 判断解析是否成功
   - 是否有错误信息

3. **尝试不同波特率**:
   - 默认: 19200
   - 备选: 9600, 38400, 57600

## 📝 技术细节

### 数据流程
```
设备输出 
  ↓
串口读取 (read_data)
  ↓
缓冲区处理 (处理分包问题)
  ↓
数据解析 (parse_data)
  ↓
信号发送 (data_received.emit)
  ↓
界面更新 (refresh_display)
  ↓
仪表盘显示 + 曲线绘制
```

### 支持的数据格式
1. **带标签格式**: `Tdf= 6.26 'C Tdfatm= 6.45 'C H2O= 9611 ppm`
2. **纯数字格式**: `19.8 23586.2 20.0` (Tdf H2O Tdfatm)
3. **部分标签格式**: `Tdf=23.20 H2O=29249`

### 关键参数
- 波特率: 19200 (可配置)
- 数据刷新间隔: 500ms (可配置)
- 历史数据最大点数: 1000 (可配置)
- 最大重连次数: 3
