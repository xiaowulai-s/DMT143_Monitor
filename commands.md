## 常用命令列表

### 基本控制命令

| 命令 | 功能说明 | 示例响应 |
|------|---------|---------|
| `?` | 帮助/显示菜单 | 设备信息列表 |

### 输出格式设置 (FORM命令)

| 命令 | 功能说明 |
|------|---------|
| `FORM TDF TDFA H2O` | 设置输出格式：露点+标准气压露点+水含量 |

| `FORM H2O` | 只输出体积含水量 |
| `FORM` | 查询当前格式设置 |

**重要**: 使用空格分隔参数，不要使用逗号。

## 数据输出格式

### 官方标准格式 (带标签)

根据DMT143官方文档，设备输出格式如下：

```
Tdf= 17.75 'C Tdfatm= -5.25 'C H2O= 954 ppm
Tdf= 17.23 'C Tdfatm= -5.72 'C H2O= 930 ppm
```

**参数顺序**: Tdf → Tdfatm → H2O

**说明**:
- `Tdf` - 露点温度 (Dewpoint)
- `Tdfatm` - 标准气压露点 (Dewpoint at atmospheric pressure)
- `H2O` - 体积含水量 (Water vapor content)
- 温度值后面可能有单位 `'F` (华氏度) 或 `'C` (摄氏度)
- H2O单位是 `ppm`

### 简化格式 (纯数字)

根据实际设备测试，简化格式的顺序为：

```
20.3 24328.9 20.5
```

**实际解析顺序**: Tdf | H2O | Tdfatm

- 第一个值: Tdf (露点温度，通常在 -100 ~ 20 °C 范围)
- 第二个值: H2O (水含量，通常 > 100 ppm)
- 第三个值: Tdfatm (标准气压露点，接近Tdf)

### 物理关系

在标准大气压下，满足: `Tdfatm >= Tdf`

### 混合格式

设备可能输出部分参数：
```
Tdf=23.20 H2O=29249
Tdf=23.40
```

## 数据参数说明

| 参数 | 符号 | 单位 | 范围 | 说明 |
|------|------|------|------|------|
| 露点温度 | Tdf | °C | -100 ~ 20 | 低温露点 |
| 标准气压露点 | Tdfatm | °C | -100 ~ 20 | 常压露点温度 |
| 体积含水量 | H2O | ppm | 0 ~ 50000 | 百万分比浓度 |
| 相对湿度 | RH | %RH | 0 ~ 100 | 相对湿度(需设备支持) |
| 过程温度 | T | °C | -40 ~ 60 | 传感器温度(需设备支持) |

## 推荐使用流程

1. **连接设备**: 打开串口 (波特率19200, 8N1)
2. **停止输出**: 发送 `S` 命令
3. **查询信息**: 发送 `I` 命令获取设备信息
4. **设置格式**: 发送 `FORM TDF TDFA H2O`
5. **设置间隔**: 发送 `INTV 1S` (可选)
6. **开始采集**: 发送 `R` 命令开始连续输出

### 完整初始化序列

```
S           ; 停止当前输出
FORM TDF TDFA H2O   ; 设置输出格式
INTV 1S     ; 设置输出间隔为1秒
SMODE RUN   ; 设置为运行模式
R           ; 开始输出
```

## 注意事项

1. **半双工通信**: 设备使用半双工通信，发送命令前建议先发送 `S` 停止输出
2. **命令响应时间**: 某些命令需要设备处理时间，建议等待0.5秒以上
3. **输出格式**: 不同设备配置可能输出不同格式，需要兼容多种格式
4. **物理约束**: 在标准大气压下，Tdfatm >= Tdf
5. **H2O范围**: H2O ppm值通常在100-50000之间

## 故障排除

### 无法连接
- 检查串口线连接
- 确认设备已通电
- 检查波特率设置(默认19200)
- 确认端口没有被其他程序占用

### 数据解析错误
- 使用 `FORM TDF TDFA H2O` 确保输出完整数据
- 检查是否有多余的换行符或特殊字符
- 尝试使用简化格式 `FORM` 查询当前格式

### 数据异常
- 检查物理关系: Tdfatm 应该 >= Tdf
- H2O ppm值应该在合理范围内
- 考虑环境温度和压力影响
