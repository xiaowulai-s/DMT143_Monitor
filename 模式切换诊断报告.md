# DMT143 模式切换诊断报告

## 测试日期
2026-03-19

## 问题总结
设备无数据输出，尝试切换通信模式

## 测试结果

### 1. ASCII 命令测试
| 命令类型 | 命令 | 结果 |
|---------|------|------|
| 停止输出 | S | ✅ 返回 0x00 (唯一响应) |
| 运行模式 | SMODE RUN | ❌ 无响应 |
| 轮询模式 | SMODE POLL | ❌ 无响应 |
| 设备信息 | I, ?, HELP | ❌ 无响应 |
| 格式设置 | FORM | ❌ 无响应 |
| 模式切换 | EXIT, MODE ASCII | ❌ 无响应 |
| 开始输出 | R, SEND | ❌ 无响应 |

### 2. Modbus RTU 测试
- **从站扫描**: 扫描 1-247，未发现任何活跃从站
- **常用地址测试**: 1, 2, 240, 247 都无响应
- **寄存器读取**: 功能码 03, 04 都无有效响应

### 3. 广播命令
- 广播地址 0xFF 有响应（返回 0x00）
- 这是唯一能获得响应的地址

## 关键发现

### S 命令的特殊性
- **S 命令返回 0x00** 是唯一能获得响应的命令
- 这可能是：
  1. **停止确认码** - 设备确实在某种模式下响应
  2. **线路回环** - 但其他数据测试排除了简单回环
  3. **广播响应** - S 可能作为广播停止命令工作

### 设备状态
- 屏幕无显示
- 无面板按键
- RS485接线: A接A, B接B

## 可能原因分析

### 1. 设备供电问题 (最可能)
- DMT143 需要外部 24V DC 供电
- 屏幕无显示表明主电源可能未接通
- RS485 接口可能有独立供电或处于待机状态

### 2. RS485 接线问题
- 当前 A-A, B-B 直连
- 不同厂商定义可能相反，建议尝试 **交叉连接**
- A 接 B，B 接 A

### 3. 设备处于特殊模式
- Modbus RTU 模式（但测试未发现从站）
- 休眠/保护模式
- 需要特定唤醒序列

### 4. 设备配置问题
- 可能需要 Vaisala Insight 软件配置
- 可能有硬件跳线/DIP 开关
- 固件可能需要更新

## 建议操作

### 立即尝试
1. **检查供电**
   - 确认 24V DC 电源已连接
   - 检查电源极性
   - 测量电压

2. **交换 RS485 线**
   - 将 A 和 B 线交叉连接
   - 重新测试

3. **断电重启**
   - 断开电源 10 秒
   - 重新上电
   - 立即运行测试

### 后续方案
1. 使用 **Vaisala Insight** 官方软件
2. 检查设备是否有 **DIP 开关** 或 **跳线**
3. 联系 **Vaisala 技术支持**
